Compétence Technique

🔀 Git

Outil central de gestion de versions utilisé dans tous mes projets académiques et professionnels pour structurer le travail et collaborer efficacement.

Ma définition

Git est un système de gestion de versions indispensable dans le développement logiciel moderne. Il permet de suivre l'évolution du code, de collaborer efficacement, de garantir la traçabilité des modifications et de sécuriser les livrables. Dans mes projets académiques comme professionnels, Git a toujours été un outil central pour structurer mon travail, organiser mes développements et assurer la stabilité des différentes versions du projet.

Mes éléments de preuve

J'ai utilisé Git dans mes projets de fin d'études ainsi que durant mon alternance, où il m'a servi aussi bien pour organiser mon travail individuel que pour faciliter mes échanges avec l'équipe. Dans certains projets, notamment lors du développement de l'application mobile et dans mes outils en alternance, Git m'a permis de gérer efficacement les différentes branches, d'expérimenter de nouvelles fonctionnalités sans risque, et de produire un historique clair et propre.

Au-delà de la simple sauvegarde, Git a été un support essentiel pour structurer ma manière de travailler, documenter mes avancées et assurer la continuité du développement.

Preuve de compétence et valeur ajoutée

L'usage de Git m'a permis de produire un code mieux organisé et plus fiable, d'éviter les régressions, et de faciliter les validations intermédiaires avec mes encadrants ou les équipes métier lorsque je travaillais en alternance. Grâce à un versionnement rigoureux (commits clairs, branches structurées, merges maîtrisés), j'ai pu fluidifier la communication technique, suivre précisément les évolutions du projet et sécuriser chaque étape du développement.

Cette compétence a renforcé ma capacité à travailler proprement, à gérer plusieurs tâches en parallèle et à reprendre facilement du code plusieurs semaines plus tard, sans perte d'information.

Mon autocritique

Niveau actuel, recul et marge de progression

Je maîtrise aujourd'hui les usages essentiels et avancés de Git : gestion des branches, rebase, résolution de conflits, suivi des issues, bonnes pratiques de commit, et structuration d'un workflow clair.

Avec le recul, Git est devenu un outil naturel dans ma pratique quotidienne. Je souhaite encore progresser sur des workflows collaboratifs plus poussés (GitFlow, trunk-based development) et optimiser la qualité de mes revues de code, même en environnement individuel.

Niveau actuel90%

Vitesse d'acquisition

Git s'est appris naturellement, au fil de mes projets académiques puis professionnels. Le fait de l'utiliser dès le début de chaque projet, y compris en solo, a accéléré considérablement ma progression. En le pratiquant quotidiennement dans un contexte réel (gestion de branches, résolution de conflits, suivi de versions), j'ai rapidement atteint un niveau opérationnel.

Conseils pour moi-même et les autres

Je conseille vivement d'utiliser Git dès le premier projet, même seul et même simple, le réflexe du commit régulier et du message clair est une habitude qui s'acquiert tôt et qui fait toute la différence en équipe. Il faut également apprendre à lire l'historique, comprendre les branches et ne pas avoir peur de faire des erreurs : Git est conçu pour permettre de revenir en arrière.

Mon évolution dans cette compétence

Place dans mon projet professionnel et objectif à moyen terme

Git est une compétence incontournable dans mon futur métier de développeuse. À moyen terme, je veux être capable de gérer et structurer des dépôts complexes, de collaborer sur des projets multi-équipes. Mon objectif est de faire de Git un outil que je maîtrise non seulement techniquement, mais aussi méthodologiquement, afin d'améliorer l'efficacité collective et la qualité des projets auxquels je participerai.

Formations et autoformations

Pour continuer à progresser, je prévois de suivre des formations courtes sur les workflows collaboratifs avancés et les bonnes pratiques professionnelles de versionnement. Je réalise également une veille régulière, via la documentation officielle et des tutoriels spécialisés, afin d'améliorer mes habitudes, d'adopter des standards plus professionnels et d'utiliser Git de manière encore plus fluide au quotidien.
