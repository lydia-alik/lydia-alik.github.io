Réalisation 05 — Projet académique

✈️ Site de réservation de billets d'avion

Plateforme web complète développée avec Spring Boot, React et PostgreSQL, une application full-stack moderne simulant un vrai service de réservation de billets d'avion.

Spring BootReact PostgreSQLAPI RESTGit

Présentation

Dans le cadre d'un projet d'école réalisé en binôme, j'ai participé au développement d'une plateforme web de réservation de billets d'avion. L'objectif était de concevoir une application complète permettant aux utilisateurs de consulter des vols, rechercher des destinations, créer un compte, effectuer des réservations et gérer leurs informations.

Le projet reposait sur une architecture moderne utilisant Java Spring Boot pour le backend, React pour le frontend, et une base de données relationnelle PostgreSQL pour la gestion des informations (utilisateurs, vols, réservations…). Ce projet avait pour vocation de simuler un service complet comparable à une plateforme de réservation réelle, tout en respectant des contraintes de cohérence, de sécurité et de fluidité d'expérience utilisateur.

Objectifs

  • Concevoir une plateforme ergonomique permettant aux utilisateurs de consulter facilement les vols disponibles et d'effectuer leurs réservations
  • Mettre en place une architecture fiable reposant sur une séparation claire entre le front-end et le back-end, afin d'assurer la maintenabilité et l'évolution future de l'application
  • Gérer l'ensemble du cycle utilisateur : création de compte, authentification, recherche de vols, sélection et validation de la réservation
  • Garantir l'intégrité et la cohérence des données stockées dans la base PostgreSQL, notamment pour éviter les incohérences dans la gestion des réservations et des disponibilités

Contexte

Projet réalisé dans un cadre académique avec un enseignant pour le suivi et une échéance fixe. Nous devions démontrer la maîtrise d'un stack technique complet : Spring Boot, React, API REST, gestion d'une base relationnelle et intégration de toutes les couches.

Enjeux

  • Concevoir un modèle de données cohérent, capable de gérer un volume potentiellement important d'informations liées aux vols, aux utilisateurs et aux réservations, sans risque d'incohérence
  • La mise en place d'une API REST fiable et sécurisée, garantissant des échanges fluides et robustes entre le front-end React et le back-end Spring Boot
  • Offrir une expérience utilisateur simple, intuitive et agréable, afin de permettre à chacun de rechercher un vol, de consulter les informations essentielles et de finaliser une réservation sans friction

Risques

  • Pour le projet : difficultés d'intégration entre les différentes couches, incohérences dans le modèle de données, retards cumulés en cas de mauvaise organisation
  • Pour notre binôme : nécessité de répartir les tâches efficacement, risque de surcharge si un membre avance moins vite, gestion du temps imposée par l'école

Étapes du projet

1

Spécification fonctionnelle

Nous avons commencé par une spécification fonctionnelle, listant les pages nécessaires, les fonctionnalités prioritaires, et les interactions entre les utilisateurs et le système. Une première version du schéma de base de données a été conçue pour représenter les vols, les compagnies, les utilisateurs, les réservations et les relations entre ces entités.

2

Développement backend (Spring Boot)

Une fois la structure validée, nous avons développé le backend en Java Spring Boot, avec mise en place des entités, des endpoints REST, de la gestion de la sécurité, et des opérations CRUD.

3

Développement frontend (React)

En parallèle, nous construisions l'interface frontend en React, tout en assurant le lien avec l'API via des appels HTTP.

4

Intégration et tests

Nous avons ensuite procédé à l'intégration complète du système : tests d'affichage de vols réels depuis la base PostgreSQL, création de comptes et réservations, vérification de la cohérence des données. Enfin, une phase de tests et de validation a permis de vérifier la sécurité, les règles métiers (impossibilité de réserver deux fois la même place, cohérence des dates…), et la bonne communication entre les différentes couches. La documentation a été rédigée pour présenter l'architecture, le modèle de données, les fonctionnalités et les choix techniques.

Acteurs

  • Mon binôme et moi : nous avons travaillé ensemble pour répartir efficacement les tâches entre frontend, backend et gestion de la base, tout en maintenant une communication fluide
  • L'encadrant : nous guidait lors des points hebdomadaires, validait nos choix techniques et nous aidait à rediriger nos efforts lorsque certaines pistes étaient trop complexes ou trop longues à implémenter

Pour le projet

Pour le projet, nous avons livré une application fonctionnelle, stable et cohérente, répondant aux principales attentes : consultation de vols, création de comptes, réservations et gestion des données. Ce projet a permis de valider l'ensemble des compétences attendues dans notre cursus.

Pour moi

Pour moi, cette réalisation a été l'occasion de renforcer mes compétences en Spring Boot, React, PostgreSQL, API REST, modélisation de données, ainsi qu'en gestion de projet en binôme. J'ai également développé une bonne compréhension des enjeux liés à la cohérence des données et au fonctionnement des applications complètes modernes.

Lendemains du projet

  • Futur immédiat : utiliser ce projet comme référence pour structurer d'autres applications en architecture front / back séparée. Intégrer les bonnes pratiques acquises (authentification, API, gestion d'erreurs).
  • À moyen terme : améliorer mes compétences sur des sujets avancés comme l'optimisation SQL, la montée en charge, les systèmes distribués, ou encore le déploiement de plateformes complètes (CI/CD, conteneurisation…)
  • Aujourd'hui : ce projet constitue une base solide pour mes projets actuels et futurs. Il illustre ma capacité à concevoir une application web complète, à interconnecter plusieurs technologies modernes et à gérer efficacement un projet de développement logiciel.

Mon regard critique

Ce projet a été particulièrement formateur. Il m'a permis de prendre du recul sur l'importance de la structuration des données et de l'architecture logique entre les couches système. Ma valeur ajoutée a résidé dans ma capacité à structurer le modèle de données, à assurer l'intégration entre backend et frontend, et à maintenir une organisation stable pour garantir l'avancement du projet.

Les principaux enseignements que j'en retiens sont : l'importance d'une modélisation claire, la nécessité de tester en continu, et le rôle fondamental de la communication au sein d'un binôme pour garantir une cohérence globale.
